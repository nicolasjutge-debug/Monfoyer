⚠️ Important sur la sauvegarde

La sauvegarde automatique ("Sauvegarder ces paramètres", les relevés) utilise une mémoire propre à claude.ai. Elle fonctionne uniquement quand ce fichier est ouvert dans l'appli/le site Claude (artefact). Si vous téléchargez le .html et l'ouvrez directement dans Safari/Chrome sur PC ou iPhone (double-clic, hors claude.ai), cette mémoire n'existe pas → message "Erreur de sauvegarde". C'est pour ça que rien ne se retrouve d'un appareil à l'autre : ce n'est pas un bug de vos réglages, c'est une limite de la sauvegarde locale elle-même. La synchronisation JSONBin ci-dessous est justement faite pour contourner ça, car elle passe par internet plutôt que par cette mémoire locale.

Synchronisation JSONBin

Synchro automatique entre appareils via jsonbin.io
Pas encore synchronisé.
La clé et le Bin ID sont mémorisés dans ce fichier tant que la page reste ouverte. Notez-les à part (copiez-les) : c'est le Bin ID + la clé, identiques sur PC et iPhone, qui permettent aux deux appareils de lire les mêmes données. Cette synchro passe par internet (fetch), donc fonctionne aussi bien dans claude.ai que dans un navigateur classique.

Utiliser sur un autre appareil (PC)

La sauvegarde automatique reste liée à cet appareil. Pour retrouver vos données sur un PC, deux façons de faire :
1. Le plus simple — envoyez-vous le fichier .html (mail, clé USB, cloud) et ouvrez-le dans un navigateur sur le PC. Utilisez ensuite "Exporter" ci-dessous sur l'appareil qui a les données, puis "Importer" sur le PC.

2. Export / Import manuel — copiez le code ci-dessous et collez-le dans la case Import sur l'autre appareil.
